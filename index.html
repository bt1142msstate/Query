    Help Quick guide

    Start Here

    Build a report in a few clear steps.

    Add the fields you want, choose filters, run the query, then review, share, reload, or export the results. This guide focuses on the controls people use most often.

    Build Add display columns and filters from the same field picker.
    Run Use Run and watch progress while results load.
    Finish Use history, templates, sharing, JSON, or Excel export when the table looks right.
    Quick Start

    The normal workflow

    1. 1
      Add fields.

      Use Add Field or the builder search to choose columns, filters, or both.

    2. 2
      Set filter values.

      Use the Display & Filters panel to edit, reorder, split, or remove fields.

    3. 3
      Run the query.

      Press Run. While it runs, the progress view shows elapsed time, backend progress, and loaded rows when available.

    4. 4
      Review and reuse.

      Sort, resize, post-filter, share results, export, save a template, or reload the run from history.

    Fields

    Choosing fields

    • Field-name matches rank ahead of description matches, so exact names rise to the top.
    • Some backend-defined fields ask for details before becoming real display columns, such as MARC tag and optional subfield inputs.
    • The available fields come from backend metadata, so deployments can define their own field list, builders, warnings, and filter support.
    Filters

    Filtering

    • Backend filters decide what gets requested from the API.
    • Post filters only refine rows already loaded in the table and never get sent to the backend.
    • Post filters clear between query runs so one report does not silently affect the next.
    • Post filters handle blank values, values that exist, and multi-value cells.
    • Date filters support normal date comparisons and Never values.
    Table

    Using results

    • Drag headers to reorder columns and resize headers when the table needs more room.
    • Right-click on desktop or long-press on mobile for cell and column actions.
    • Ellipsized and multi-value cells stay compact by default; click them to inspect or copy the full values.
    • Split mode can expand multi-value fields into grouped columns without changing the original result data.
    • Use expand mode when the table needs the full screen, including on mobile.
    Controls

    Top buttons

    Run

    Runs the current query. While active, the button switches to stop mode.

    Export

    Builds an Excel workbook with progress shown during large exports.

    Split

    Toggles multi-value fields between compact cells and split columns.

    JSON

    Shows the backend payload for the current query.

    History

    Shows running, completed, failed, and canceled query runs.

    Templates

    Creates, edits, pins, categorizes, and reloads reusable query setups.

    Sharing

    Form and result links

    • Share copies a limited link, and when results are loaded it defaults to a result link.
    • Form Link copies a clean editable form link without opening saved results.
    • Refresh keeps the current editable form or opened results instead of forcing limited mode.
    • Old URL flags are cleaned into the current form, limited, tableName, and result parameters.
    History

    Recover or repeat work

    Useful actions

    • Reload a previous query setup.
    • Open results from a completed run or partial results from a run that is still active.
    • Cancel a run that is still active.
    • Rerun a useful report without rebuilding it.
    • Refresh returns to the last opened results when the result URL or local cache is available.

    Statuses

    • RunningStill executing on the backend.
    • CompleteFinished and ready to load.
    • FailedStopped because something went wrong.
    • CanceledStopped intentionally before finishing.
    Tips

    Small habits that help

    Check JSON when unsure.

    It shows the exact backend payload.

    Use templates for repeat reports.

    Save setups you expect to run again.

    Confirm export settings first.

    Column order, grouping, split mode, and optional run details affect the workbook.

    Connect your own API for production.

    The live site is a frontend; compatible backend URLs can be configured per deployment.
