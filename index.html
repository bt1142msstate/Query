Query Guide

Build library reports without guessing what the tool is doing.

Start with bubbles, shape the result table, run the query, then reuse or export what worked. The interface is designed so the visual state stays aligned with the backend payload.

1. Build Choose columns and filters from bubbles.
2. Run Execute the flat backend query with the play button.
3. Review Inspect results, history, JSON, and exports.
Workflow

Recommended order

  1. 01
    Find a field.

    Use the search bar or category tabs to narrow the bubble list to the field you want.

  2. 02
    Choose how it should be used.

    Drag a bubble to the table to add a column, or click it to open filter controls for that field.

  3. 03
    Refine the query.

    Add more fields, reorder columns, and confirm filter values until the JSON matches the result shape you expect.

  4. 04
    Run, review, reuse.

    Run the query, inspect history, reload useful configurations, and export results when the table looks right.

Bubbles

How field bubbles behave

  • Click a bubble to open the filter card for that field.
  • Drag a bubble into the table area to add it as a displayed column.
  • Displayed or filtered bubbles stay visually marked so you can scan the current query quickly.
  • Some fields offer grouped lists or boolean pills because those values come from backend field metadata.
Filters

How filtering works now

  • Filtering is flat and backend-aligned. There is no AND/OR group builder in the current query model.
  • Equals filters on selector fields replace the previous selection instead of stacking conflicting values.
  • The side filter panel and the bubble panel both edit the same live filter state.
  • Panels stay open after applying values so you can keep refining a field without reopening it.
Table

Working with results

  • Drag table headers to reorder columns after you add them.
  • Remove columns from the table when they are no longer needed.
  • The split export toggle changes how multi-value cells are written when exporting results.
  • Clearing the query resets the current working state so you can start over cleanly.

History

Query history and recovery

What you can do

  • Reload a past query’s displayed columns and filters.
  • Open saved results for completed runs.
  • Cancel a query that is still running.
  • Rerun a previous query from the history panel.

Status meanings

  • RunningCurrently executing on the backend.
  • CompletedFinished successfully and ready to reload.
  • FailedStopped because of an error or interrupted backend state.
  • CancelledStopped intentionally before completion.
Practical Tips

Things that save time

Use JSON as a truth check.

If the UI feels off, open the JSON panel. It shows the real query payload the backend will receive.

Use history for repeat work.

If a report is close to what you need, reload it from query history instead of rebuilding it manually.

Prefer selectors for policy fields.

When a field offers a list or boolean pill, use it instead of typing freeform text to avoid bad literals.

Refine before exporting.

Check split export mode and column order first so the downloaded file matches how you intend to use it.
